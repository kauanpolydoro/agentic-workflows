## Registry identity

Recipe metadata uses the stable maintainer ID `project-maintainers`.

That ID resolves to the active maintainer group defined by `packages/core/src/maintainer-registry.ts`.

The current group member is GitHub user `kauanpolydoro`, with project stewardship, release, and security-coordination responsibilities.

Maintainers review compatibility evidence, release artifacts, and changes to the recipe trust model.
The linked GitHub profile establishes maintainer identity but is not a private contact channel.
Use the private vulnerability reporting process in [SECURITY.md](SECURITY.md) for sensitive security reports.

Future maintainer nominations require sustained contributions, sound judgment, and agreement from current maintainers.
