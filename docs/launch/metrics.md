# Launch metrics

The product ships no telemetry, so project health is measured from public activity and voluntary feedback only.

- Count distinct issue and discussion participants from public GitHub activity.
- Track accepted recipe and adapter contributions.
- Record documentation defects and how long they took to correct.
- Review release downloads only after GitHub releases exist.
- Ask volunteers whether a clean-clone setup and a disposable installation worked for them.
- Categorize support requests to decide which docs and product fixes matter most.

Stars may indicate awareness, but they are not a quality target and must never be purchased, traded, rewarded, or automated.
The product must not collect repository content, CLI arguments, environment variables, or user identity.
