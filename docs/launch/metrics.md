# Launch metrics

Measure project health without product telemetry or invasive tracking.

- Count distinct issue and discussion participants from public GitHub activity.
- Track accepted recipe and adapter contributions.
- Record documentation defects and time to correction.
- Review release downloads only after GitHub releases exist.
- Ask volunteers whether clean-clone setup and a disposable installation succeeded.
- Categorize support requests to prioritize docs and product fixes.

Stars may indicate awareness but are not a quality target and must never be purchased, traded, rewarded, or automated.
Do not collect repository content, CLI arguments, environment variables, or user identity through the product.
