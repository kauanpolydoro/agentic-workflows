# Agentic Workflows v0.2.1

Agentic Workflows v0.2.1 makes the path from installation to first workflow invocation explicit and strengthens synchronization between the repository, npm package pages, and GitHub releases.

## Highlights

### Complete first-use journey

The English and Brazilian Portuguese landing pages now guide users from global installation through project discovery, initialization, dry-run review, installation, agent invocation, and health checks.

The quick start uses the globally installed `awf` command and shows the expected output at the points where users commonly need confirmation.

Existing projects now have explicit configuration discovery, project-root selection, and safe reinitialization guidance.

### Package and landing-page parity

The CLI package now includes both canonical landing pages and the complete documentation needed by their packaged links.

Build preparation copies the repository landing pages into the CLI package, while delivery and package smoke tests enforce exact English and Brazilian Portuguese parity.

Exact-version examples now identify `0.2.1` for reproducible project and CI installation.

### Safer release synchronization

The release workflow now audits allowlisted public documentation links before either npm package is published.

After publication, the resumable publisher verifies both npm `dist.integrity` and the exact registry README bytes against the local package.

An existing npm version is accepted only when its package integrity and README match, which prevents a resumed workflow from silently accepting different published content.

### Accurate compatibility claims

Public compatibility documentation now distinguishes active evidence from execution records archived during the intentional Git history reset.

Archived Claude Code and Codex records remain available as historical context but no longer promote current external execution status.

The authoring guide now points to recipe schema version 3 and separates per-recipe compatibility declarations from global adapter facts.

## Added

- Brazilian Portuguese README distribution in the CLI npm tarball.
- Packaged authoring guidance and terminal demonstration assets required by README links.
- Semantic parity tests for both landing pages and their first-use commands.
- npm registry README verification in the resumable publication contract.
- External documentation-link auditing before irreversible npm publication.

## Changed

- Reorganized both landing pages around installation, initialization, preview, installation, invocation, and lifecycle maintenance.
- Added concrete expected output, troubleshooting paths, exact project pinning, and a complete contributor validation reference.
- Updated roadmap, launch, release, and security documents to describe the current trusted-publishing and pre-1.0 support state.
- Made package and acceptance smoke tests derive the expected recipe count from repository data instead of a hard-coded value.

## Fixed

- Replaced browser-specific `awf show --open` help and failure text with the operating system's native document-handler contract.
- Corrected adapter research and compatibility claims so archived evidence cannot be interpreted as current execution coverage.
- Corrected stale schema and installation guidance in linked onboarding documents.

## Breaking changes

No intentional breaking changes are included in v0.2.1.

Existing commands, JSON contracts, installation manifests, and workflow invocation formats remain compatible with v0.2.0.

## Upgrade

Update the globally installed CLI:

```bash
npm install --global @kauanpolydoro/agentic-workflows@0.2.1
```

Confirm the selected version and inspect the current project:

```bash
awf --version
awf context
awf doctor
awf status
```

For a project-local or CI installation, pin the exact version:

```bash
npm install --save-dev --save-exact @kauanpolydoro/agentic-workflows@0.2.1
npx awf list --json
```

## Requirements and compatibility

- Node.js 22 or newer is required.
- Normal CLI operation remains offline and does not send telemetry.
- The package exposes both `awf` and `agentic-workflows`, with `awf` used throughout the documentation.
- The unscoped `agentic-workflows` package on npm is a different project.

## Verification boundary

Repository validation covers schemas, generation, local and packed installation, CLI automation, terminal acceptance, cross-platform behavior, documentation, and release contracts.

This release does not claim that an external agent executed every workflow or that a human reviewed every generated outcome.
