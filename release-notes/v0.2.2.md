# Agentic Workflows v0.2.2

Agentic Workflows v0.2.2 is a packaging-only hotfix that restores exact landing-page parity between GitHub and npm.

## What happened in v0.2.1

The v0.2.1 CLI tarball contained both `README.md` and `README.pt-BR.md` at its root.

npm selected `README.pt-BR.md` as the package landing page even though `README.md` was the intended canonical source.

Both v0.2.1 npm packages were published successfully with provenance, but the new post-publication README verifier detected the mismatch and stopped the workflow before a GitHub Release was created.

The CLI itself remained functional, and no runtime or workflow data was corrupted.

## Fixed

- Kept exactly one root README named `README.md` in each npm tarball.
- Removed the translated README from the CLI package root so it cannot compete for npm landing-page selection.
- Kept the Brazilian Portuguese landing page public in the GitHub repository and linked it directly from the canonical English README.
- Added a package smoke check that rejects zero, multiple, or incorrectly named root README files before publication.
- Extended registry verification to require both exact README bytes and `readmeFilename: README.md` after publication.

## Compatibility

No CLI runtime, command, JSON contract, recipe, adapter, or installation-manifest behavior changed in v0.2.2.

The core and CLI packages remain versioned together, so the core package is also published as v0.2.2 without behavioral changes.

No intentional breaking changes are included.

## Upgrade

Update the globally installed CLI:

```bash
npm install --global @kauanpolydoro/agentic-workflows@0.2.2
```

Confirm the selected version:

```bash
awf --version
awf doctor
```

For a project-local or CI installation, pin the exact version:

```bash
npm install --save-dev --save-exact @kauanpolydoro/agentic-workflows@0.2.2
npx awf list --json
```

## Requirements

- Node.js 22 or newer is required.
- Normal CLI operation remains offline and does not send telemetry.
- The package exposes both `awf` and `agentic-workflows`, with `awf` used throughout the documentation.
